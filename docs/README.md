# Project Documentation

This folder explains architecture, engineering standards, operations, and adoption practices that
are broader than one source folder or component.

## Contents

- `NAMING_CONVENTIONS.md` defines naming rules.
- `TESTING.md`, `LOGGING.md`, and `PERFORMANCE_MONITORING.md` describe engineering practices.
- `BRANCHING_STRATEGY.md`, `COMMIT_MESSAGES.md`, and `NPM_PUBLISHING.md` describe delivery.
- `adr/` records important architectural decisions and their reasoning.

## How to use

Read the relevant guide before changing a cross-cutting concern. Update an existing guide when its
policy changes. Add an ADR when choosing between meaningful architectural alternatives that future
contributors may otherwise revisit.

Folder READMEs explain where code belongs; these guides explain the standards and reasoning behind
the code.
