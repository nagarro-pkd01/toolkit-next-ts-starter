# Claude Skills Guide

## Skill Design Recommendations

- Frame skills around workflows instead of generic tips
- Include project-specific constraints and expected outputs
- Define explicit checks before marking a task complete

## Suggested Skill Library

- `architecture-review`: evaluate atomic design boundaries and config ownership
- `change-safety`: assess migration risk, compatibility, and rollback options
- `release-coordinator`: ensure conventional commits and semantic-release compatibility

## Governance

- Document skill owners and update frequency
- Track confidence notes for each skill output
- Retire skills that no longer map to active engineering workflows
