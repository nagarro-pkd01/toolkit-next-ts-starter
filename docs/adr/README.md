# Architecture Decision Records

Architecture Decision Records (ADRs) preserve why a significant technical decision was made.

## How to use

Create a numbered Markdown file such as `0002-client-side-logging.md` with:

1. Context and problem
2. Considered options
3. Decision
4. Consequences and trade-offs
5. Status such as proposed, accepted, superseded, or rejected

ADRs should document decisions, not implementation tutorials. Do not rewrite an accepted ADR to
hide history; add a new ADR that supersedes it.
